# 🏏 IPL Data Analytics & Match Winner Prediction System

An end-to-end Data Analytics + Machine Learning + BI project covering data cleaning, exploratory
data analysis, predictive modeling, an interactive Streamlit app, and a Power BI dashboard —
built on IPL (Indian Premier League) match and ball-by-ball data.

> **Note on data**: This repo ships with a realistic **synthetic IPL dataset** (10 teams, 7
> seasons, 400+ matches, ~96K deliveries) generated by `data/generate_data.py` so the whole
> pipeline runs out of the box. To use **real IPL data**, download the "IPL Complete Dataset"
> from Kaggle (`matches.csv`, `deliveries.csv`) and drop them into `data/` with the same column
> names — the cleaning, EDA, ML, and app code will work unchanged.

## ⚙️ Installation Guide

```bash
# 1. Clone the repository
git clone https://github.com/<your-username>/ipl-analytics-project.git
cd ipl-analytics-project

# 2. Create virtual environment
python -m venv venv
venv\Scripts\activate          # Windows (PowerShell)
# source venv/bin/activate     # macOS/Linux

# 3. Install dependencies
pip install -r requirements.txt

# 4. (Optional) Re-generate synthetic data
python data/generate_data.py

# 5. Run data cleaning + feature engineering
python data/data_cleaning.py

# 6. Run EDA (generates charts in src/eda_outputs/)
python src/eda.py

# 7. Train ML models (saves model.pkl, meta.pkl)
python src/ml_model.py

# 8. Launch the Streamlit app
streamlit run streamlit_app/app.py
```

## 🔍 1. Data Collection

Five core datasets:
- **matches.csv** – match-level info (teams, venue, toss, result, scores)
- **deliveries.csv** – ball-by-ball data (batsman, bowler, runs, wickets)
- **players.csv** – player roles, batting/bowling style, nationality
- **teams.csv** – team metadata (home venue, founded year)
- **venues.csv** – venue metadata (city, capacity)

## 🧹 2. Data Cleaning (`data/data_cleaning.py`)
- Standardized inconsistent team names (e.g. `Punjab Kings (PBKS)` → `Punjab Kings`,
  `Delhi Daredevils` → `Delhi Capitals`)
- Filled missing `city` values from venue mapping
- Dropped duplicate match rows
- Converted `date` to datetime, extracted `year`/`month`
- Engineered features: `toss_match_winner_same`, `run_diff`, `batting_first_won`,
  `target_score`, `crr` (current run rate), `rrr` (required run rate), `runs_left`,
  `balls_left`, `wickets_left`, `is_high_scoring`

## 📊 3. Exploratory Data Analysis (`src/eda.py`)
24 charts covering:

**Player Performance** – Top Run Scorers, Orange Cap by season, Top Wicket Takers,
Purple Cap by season, Strike Rate Leaders, Economy Rate Leaders, Player Consistency

**Team Performance** – Most Successful Teams, Win %, Home vs Away, Toss Impact,
Head-to-Head Heatmap

**Venue Analysis** – Highest Scoring Grounds, Avg 1st Innings Score, Best Chasing
Grounds, Toss Advantage by Venue, Venue Win Trends

**Season Analysis** – Season Champions, Season-wise Runs/Wickets, Team Performance
Evolution

**Statistical** – Correlation Matrix, Scatter Plot (1st vs 2nd innings score),
Box Plot (score distribution by venue)

All charts saved as PNGs to `src/eda_outputs/`.

## 🤖 4. Machine Learning — Match Winner Prediction (`src/ml_model.py`)

Predicts whether the **chasing (batting second) team wins**, using live match-state
features: `batting_team`, `bowling_team`, `venue`, `target_score`, `current_score`,
`overs_completed`, `current_wickets`, `runs_left`, `balls_left`, `wickets_left`, `crr`, `rrr`.

### Model Comparison Results

| Model               | Accuracy | Precision | Recall  | F1 Score | ROC-AUC |
|---------------------|---------:|----------:|--------:|---------:|--------:|
| **XGBoost** ⭐       | 0.9991   | 0.9993    | 0.9989  | 0.9991   | 1.0000  |
| Random Forest       | 0.9684   | 0.9795    | 0.9538  | 0.9665   | 0.9974  |
| Logistic Regression | 0.8182   | 0.8173    | 0.7978  | 0.8074   | 0.9083  |

> **Best Model: XGBoost** — selected and saved as `streamlit_app/model.pkl`
>
> Note: scores are near-perfect because the synthetic data closely follows a
> deterministic chase formula. On real IPL data, expect realistic accuracy in the
> ~75-85% range, with XGBoost typically still leading.

## 🌐 5. Streamlit Web App (`streamlit_app/app.py`)
- Dark theme with IPL gold/navy color palette
- Team, venue, target/current score, overs, and wickets inputs
- "Predict Winner" button → win probability bar + donut chart (Plotly)
- Responsive 3/4-column layout with stat cards

Run locally:
```bash
streamlit run streamlit_app/app.py
```

## 🚀 Deployment

### GitHub
```bash
git init
git add .
git commit -m "IPL Data Analytics & Match Winner Prediction System"
git branch -M main
git remote add origin https://github.com/<your-username>/ipl-analytics-project.git
git push -u origin main
```

### Streamlit Cloud
1. Push the repo to GitHub.
2. Go to [share.streamlit.io](https://share.streamlit.io) → "New app".
3. Select repo, branch `main`, file path `streamlit_app/app.py`.
4. Click **Deploy**.

### Render
1. Create a new **Web Service** on [render.com](https://render.com), connect your GitHub repo.
2. Build command: `pip install -r requirements.txt`
3. Start command:
   ```
   streamlit run streamlit_app/app.py --server.port $PORT --server.address 0.0.0.0
   ```
4. Deploy.
